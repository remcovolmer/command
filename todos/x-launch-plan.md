# X Launch Plan - Command

## Doel
Bereik creëren voor Command (desktop app voor Claude Code sessies) via X/Twitter.

---

## Fase 1: Fundament (Week 1)

- [ ] **Profiel optimaliseren**
  - Bio: duidelijk wat je bouwt ("Building Command — a desktop app for managing Claude Code sessions")
  - Link naar GitHub/website
  - Pinned post: korte demo video van de killer feature

- [ ] **10 accounts identificeren om mee te engagen**
  - Zoek op: `claude code`, `ai coding`, `cursor`, `terminal workflow`, `developer tools`
  - Mix van: AI dev tool makers, power users, dev influencers
  - Volg ze, zet notificaties aan

- [ ] **2 weken lang alleen reageren, niet zenden**
  - Reageer inhoudelijk op posts over AI coding workflows
  - Deel tips, ervaringen, inzichten — geen promo
  - Doel: zichtbaar worden in de juiste kringen

---

## Fase 2: Eerste content (Week 3-4)

- [ ] **"Why I built this" thread schrijven**
  - Format: 5-7 tweets
  - Hook: het probleem (6 Claude sessions, geen overzicht)
  - Bouw: wat je probeerde, wat niet werkte
  - Payoff: korte demo GIF/video
  - CTA: link naar repo of download

- [ ] **Eerste screen recording maken**
  - Max 30 seconden
  - Focus op 1 feature die pijn oplost
  - Suggesties:
    - Split view met meerdere Claude sessions
    - Auto-resume na herstart
    - Attention indicator (busy/permission/done states)
    - Terminal pool met LRU eviction

- [ ] **Technische deep-dive post**
  - Kies 1 interessant technisch probleem dat je oploste
  - Bijv: "How I detect Claude Code's state without modifying its source"
  - Hook system + polling + BiMap uitleggen in 4-5 tweets
  - Devs waarderen craft posts

---

## Fase 3: Ritme (Week 5+)

**Schema: 2-3 posts per week**

| Dag | Type | Voorbeeld |
|-----|------|-----------|
| Di | Build in public | Feature demo, bug fix story, metrics |
| Do | Technisch/opinie | Architecture decision, hot take over AI tooling |
| Za | Community | Retweet gebruiker, vraag om feedback, poll |

**Post tijden (NL):**
- 15:00-16:00 (US ochtend)
- 21:00-22:00 (US middag)

---

## Content ideeën backlog

1. Before/after: terminal chaos vs. Command overzicht
2. "Unpopular opinion: AI coding tools need better session management"
3. Feature spotlight: worktree support voor parallel development
4. Numbers post: "Week X stats — downloads, sessions created, bugs fixed"
5. Pain point post: "The 3 things that break when you run 5 Claude sessions"
6. Vergelijking: hoe andere devs het oplossen vs. Command
7. Behind the scenes: electron + node-pty + xterm.js stack keuzes
8. User feedback highlight (zodra je die hebt)

---

## Metrics om te tracken

- Impressies per post
- Profiel visits per week
- Follower groei
- Link clicks naar repo/download
- Replies en engagement rate

---

## Anti-patronen (niet doen)

- Dagelijks posten zonder kwaliteit
- Pure promo zonder waarde
- Alleen zenden, niet engagen
- Posts zonder visueel (tekst-only presteert slecht voor tools)
- Lange threads zonder hook in tweet 1
