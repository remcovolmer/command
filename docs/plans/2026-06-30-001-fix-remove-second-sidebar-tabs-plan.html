Goal Capsule

Objective
Verwijder de dubbele Files/Git/Tasks/Auto tab-rij uit de file-explorer-flyout. De rail wordt de enige tab-switcher. Verplaats de live tellers naar de rail-iconen; behoud de refresh-knop en het branch-label in een slank flyout-header.
Authority
De units hieronder zijn leidend. Bij conflict met repo-conventies (lint, types, patterns) winnen de conventies; meld een echte tegenspraak in plaats van te gokken.
Stop conditions
Geen dubbele tab-rij meer; rail-iconen tonen git/tasks/auto-counts (verborgen bij 0); refresh + branch-label behouden; npm test en npm run build groen.
Execution profile
Standard, 3 units, één PR-waardig. Geen migratie, geen IPC-wijziging, geen nieuwe hotkeys.
Tail
Implementer draait tests/build/lint en levert op via de gebruikelijke commit/PR-flow van de repo.

Product Contract

Summary

De #141-redesign verplaatste tab-switching naar de icoon-rail uiterst rechts, maar liet de oude FileExplorerTabBar in de flyout staan. Daardoor verschijnen Files/Git/Tasks/Auto nu twee keer. Dit plan verwijdert die oude tab-rij; de teller-badges verhuizen naar de rail-iconen en de refresh-knop plus het Files · <branch>-label blijven behouden in een tab-loze flyout-header.

Problem Frame

De oude tab-rij doet meer dan tabs: hij draagt ook drie affordances die de rail vandaag niet heeft — live tellers (ongecommitte git-changes, open taken, ongelezen automation-runs), een handmatige refresh-knop, en het worktree-branch-label. Simpelweg de hele FileExplorerTabBar slopen verwijdert die affordances stilletjes mee. De gebruiker heeft bevestigd dat de tellers naar de rail moeten en refresh + branch-label behouden blijven.

Planning Contract

KTD1Repurpose FileExplorerTabBar tot tab-loze header, hernoem naar FileExplorerHeader

Strip de tab-<button>'s en de badge-spans; houd één slanke rij over met de actieve-paneel-titel, het · <branch>-suffix voor files, en de refresh-knop (R3, R4). Hernoemen naar FileExplorerHeader.tsx houdt de naam eerlijk — één consument (FileExplorer.tsx), dus lage churn. De titel volgt fileExplorerActiveTab; de tab-switching blijft volledig in de rail + store.

KTD2Badges op de rail hergebruiken het bestaande shell-count-pattern

De rail heeft al een absolute-positioned count-badge voor de shell-drawer. Hergebruik die span-vorm op de Git/Tasks/Automations-iconen. Git-count en task-count zijn al in de store (gitStatus[gitContextId] som van staged/modified/untracked/conflicted; tasksData[activeProjectId].nowCount) en worden via dezelfde selectors gelezen die de rail al voor shell-context gebruikt (activeWorktree?.id ?? activeProjectId).

KTD3Automation-unread verhuist naar een herbruikbare hook

De ongelezen-automation-count leeft nu als lokale state in FileExplorer (api.automation.listRuns + onRunCompleted/onRunFailed-listeners). De rail heeft hem nodig en FileExplorer niet meer. Extraheer een useAutomationUnreadCount()-hook (conventie src/hooks/useXxx.ts) zodat de telling op één plek leeft en de rail hem leest. Verwijder de dode state uit FileExplorer.

De verwijdering herverdeelt vier verantwoordelijkheden van één component over drie.

U1.Strip de tabs — flyout-header met titel, branch & refresh

Goal
Verwijder de dubbele tab-rij; behoud refresh-knop en branch-label in een tab-loze header.
Requirements
R1, R3, R4, R5
Dependencies

src/components/FileExplorer/FileExplorerTabBar.tsx → hernoem naar src/components/FileExplorer/FileExplorerHeader.tsx; src/components/FileExplorer/FileExplorer.tsx (import + gebruik, badge-props laten vervallen); test/fileExplorerHeader.test.tsx (nieuw)
Approach & patterns

Schrap in de component de tabs.map-buttons en de badge-spans. Houd de buitenste rij + de refresh-knop. Render links de actieve-paneel-titel afgeleid van activeTab (Files / Git / Tasks / Automations), met · {worktreeBranch} erachter wanneer activeTab==='files' en er een branch is (KTD1). Props krimpen tot activeTab, worktreeBranch, onRefresh, isGitLoading (voor de spin); gitChangeCount/taskNowCount/automationUnreadCount/onTabChange/showGitTab vervallen. In FileExplorer.tsx: werk de import bij, verwijder de doorgegeven badge-props; de handleRefresh- en isGitLoading-bedrading blijft staan.

Test scenarios
  • Header rendert de titel van het actieve paneel; bij files + worktree toont hij Files · <branch>.
  • Er zijn geen tab-switch-buttons meer in de header (geen Git/Tasks/Auto-knoppen).
  • Klik op refresh roept onRefresh aan; bij git + isGitLoading spint het icoon.
  • Geen badge-tellers meer in de header gerenderd.
Verification

Flyout opent via een rail-icoon zonder tweede tab-rij; refresh ververst het actieve paneel; branch zichtbaar bij files.

U2.Count-badges op de rail-iconen

Goal
Toon git-, task- en automation-tellers als badges op de Git/Tasks/Automations-iconen, verborgen bij 0.
Requirements
R2
Dependencies
U3
Files
src/components/Layout/ActivityRail.tsx; test/activityRail.test.tsx (nieuw)
Approach & patterns

Per ITEMS-entry een optionele count berekenen: Git = som van gitStatus[gitContextId] (staged+modified+untracked+conflicted), Tasks = tasksData[activeProjectId]?.nowCount ?? 0, Automations = useAutomationUnreadCount() (U3), Files = geen. Render de badge met de bestaande shell-count-span-vorm (absolute, -top-0.5 -right-0.5, bg-primary), alleen wanneer count > 0 (KTD2). Hergebruik de al aanwezige activeWorktree/activeProjectId-selectors voor de git-context.

Test scenarios
  • Git-icoon toont het aantal changes; badge verdwijnt wanneer de count 0 is.
  • Tasks-icoon toont nowCount; Automations-icoon toont de unread-count uit de hook.
  • Files-icoon krijgt nooit een badge.
  • Badge-vorm matcht het shell-count-pattern (zelfde span/stijl, geen one-edge accent).
Verification

Met openstaande changes/taken/runs verschijnen de juiste getallen op de rail; bij 0 zijn ze weg.

U3.Extraheer useAutomationUnreadCount-hook

Goal
Verplaats de ongelezen-automation-telling naar een herbruikbare hook; verwijder de dode state uit FileExplorer.
Requirements
R2 (afgeleid)
Dependencies
—
Files
nieuw src/hooks/useAutomationUnreadCount.ts; src/components/FileExplorer/FileExplorer.tsx (verwijder lokale unread-state + listeners); test/useAutomationUnreadCount.test.ts (nieuw)
Execution note: test-first op de hook — de telling-logica (filter op niet-gelezen én niet-running) is de kern; mock api.automation.listRuns en de run-events.
Approach & patterns

Verhuis het blok uit FileExplorer: listRuns(undefined, 100), filter !r.read && r.status !== 'running', en re-load op onRunCompleted/onRunFailed. De hook geeft een number terug en ruimt zijn listeners op bij unmount. FileExplorer verliest de bijbehorende useState + useEffect; de rail (U2) consumeert de hook.

Test scenarios
  • Hook telt alleen runs die niet gelezen én niet running zijn.
  • Een onRunCompleted/onRunFailed-event triggert een herberekening.
  • Listeners worden bij unmount afgemeld (geen leak).
  • FileExplorer rendert nog en bevat geen unread-state meer.
Verification

De auto-badge op de rail updatet wanneer een automation-run klaar is of faalt.

Definition of Done

  • De Files/Git/Tasks/Auto tab-rij is verwijderd; geen dubbele tabs in de flyout (R1).
  • De rail-iconen tonen live git/tasks/auto-counts, verborgen bij 0 (R2).
  • Refresh-knop en Files · <branch>-label blijven werken in de flyout-header (R3, R4).
  • Tab-switch-hotkeys en rail-highlight werken onveranderd (R5).
  • FileExplorerTabBar is hernoemd naar FileExplorerHeader; geen dode badge-/tab-code achtergebleven; de unread-state is uit FileExplorer naar de hook verplaatst.
  • npm test + npm run build + ESLint groen; geen experimentele/dode-eind-code in de diff.

System-Wide Impact

  • Tab-switch-state — ongewijzigd: rail en hotkeys sturen fileExplorerActiveTab in de store; het verwijderen van de tab-rij raakt dat pad niet (R5).
  • Automation-unread — verhuist van FileExplorer naar een gedeelde hook; functioneel identiek, andere bezitter (U3).
  • Geen IPC/persist-wijziging — puur renderer-UI; geen store-shape-, IPC- of migratie-impact.
