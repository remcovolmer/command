1. Probleem

Wat de gebruiker ziet:

  • In het command repo verschijnt maar 1 chat in het project overview, terwijl er gisteren meerdere chats waren.
  • Bij andere repos in C:\Users\RemcoVolmer\Code\ (bv. pascal_ai) verschijnen helemaal geen chats in het overview, ook al is daar gisteren in gewerkt.
  • Hook-gegenereerde titels en samenvattingen (~/.claude/session-summaries.json) zijn aanwezig en up-to-date, maar koppelen niet aan sessies.

Impact: "Recent sessions" in ProjectOverview.tsx is voor de meeste projecten leeg of incompleet. Resuming via overview is daardoor effectief stuk.

2. Root cause — twee gerelateerde bugs

Bug A  CRITICAL  Encoding mismatch met Claude Code

encodeProjectPath() in electron/main/services/SessionIndexService.ts (regels 54-56) vervangt alleen /, \ en : door -:

export function encodeProjectPath(cwd: string): string {
  return cwd.replace(/[/\\\\:]/g, '-').replace(/^-/, '')
}

Claude Code zelf vervangt echter elk niet-alfanumeriek karakter (behalve -) door -. Bewijs uit ~/.claude/projects/:

Project path op diskClaude Code dirnaamWat command zoektMatch?
C:\Users\RemcoVolmer\Code\command C--Users-RemcoVolmer-Code-command C--Users-RemcoVolmer-Code-command ok
C:\Users\RemcoVolmer\Code\pascal_ai C--Users-RemcoVolmer-Code-pascal-ai C--Users-RemcoVolmer-Code-pascal_ai 404
...\.claude (subpath) ...--claude (dot → dash) ...-.claude (dot blijft staan) 404

Geraakt zijn dus alle repos met _, . of andere speciale chars in het pad. In C:\Users\RemcoVolmer\Code\ staat:

command, deep_personality, fbto_coaching, intern_ai, intern_transcriptie, intern_zorgvergelijker, local_llm, mark_dadchelor, nn_bp_dashboard, pascal_ai, pascal_jurisprudentie, portfolio, portfolio_tracker, riverwise_marketplace, unive_klantcontact_ai, vattenfall_voorkomen, vattenfall_voorkomen_handover, VolmerEnterprise

⇒ 14 van de 18 projecten worden geraakt door deze encoding bug.

Bug B  HIGH  Worktree-sessies aggregeren niet in parent overview

Claude Code gebruikt per actieve cwd een aparte projectdirectory. Een chat gestart in command\.worktrees\fix-chat-summary-overview komt in een eigen dir terecht:

~/.claude/projects/
  C--Users-RemcoVolmer-Code-command                                          ← 2 jsonls
  C--Users-RemcoVolmer-Code-command--worktrees-fix-chat-summary-overview     ← 1 jsonl
  C--Users-RemcoVolmer-Code-command--worktrees-feat-improve-design           ← jsonls
  C--Users-RemcoVolmer-Code-command--worktrees-feat-file-editor              ← jsonls
  ...> 30 worktree-dirs voor het command project

De huidige SessionIndexService.scanJsonlFiles() kijkt alleen in de letterlijke project-dir. Dus als de gebruiker het command-project opent in Command Center, ziet ie alleen de 2 jsonls van de root — niet de tientallen worktree-sessies. Dit verklaart het "maar 1 chat"-symptoom.

Verifieerbaar: de directory C--Users-RemcoVolmer-Code-pascal-ai bevat 10 jsonls, w.o. eentje van 21 mei (gisteren). Na Bug A-fix verschijnen die direct.

3. Design / Aanpak

Fix Bug A — Encoding

Vervang regex in encodeProjectPath() door een class die alles behalve [A-Za-z0-9-] vervangt door -. Geen multi-dash collapse: Claude Code doet dat ook niet (zie C--Users-RemcoVolmer--claude met dubbele dash bij \\.).

cwd.replace(/[^A-Za-z0-9-]/g, '-').replace(/^-/, '')

Fix Bug B — Worktree-aggregatie

In scanJsonlFiles(): lees alle directories in ~/.claude/projects/ waarvan de naam start met <encoded-project>- (extra trailing dash). Parse jsonls uit de root-dir Én de matchende worktree-dirs in één cache.

Voordeel: geen afhankelijkheid van WorktreeService — we vinden ook sessies van inmiddels verwijderde worktrees, en het werkt voor mensen die git worktree buiten Command Center om gebruiken.

Waarom geen "scan alle dirs"?

Alternatief was: scan altijd ~/.claude/projects/ volledig, filter op cwd-prefix in elke jsonl. Verworpen — te duur (1000-en jsonls bij power users), bovendien staat de cwd niet altijd in elke jsonl-regel. Prefix-match op dirnaam is goedkoop en deterministisch.

False-positive risico bij prefix-match

command ⇒ prefix C--Users-RemcoVolmer-Code-command-. Match: C--Users-RemcoVolmer-Code-command-mvp als de gebruiker ooit een ander project command-mvp had. Niet kritisch (de jsonl-content bepaalt of het écht een command-sessie is), maar we filteren strenger: alleen dirs die na de prefix beginnen met -worktrees- of --worktrees- (Claude's encoding van /worktrees resp. /.worktrees) tellen mee.

4. Implementation Units

U1

Fix encodeProjectPath() — volledige Claude Code-encoding

Goal Path-encoding matcht 1:1 met Claude Code's dirnaam-conventie.
Files
  • electron/main/services/SessionIndexService.ts — regels 54-56
  • test/sessionIndexService.test.ts — encoding-tests toevoegen
Approach
  1. Vervang regex door /[^A-Za-z0-9-]/g.
  2. Vervolg-call .replace(/^-/, '') blijft staan (strip eventuele leading dash voor Unix-paden).
  3. Voeg JSDoc toe met expliciete verwijzing naar Claude Code's encoding-regel en voorbeelden.
  4. Check of encodeProjectPath elders in de codebase wordt gebruikt — gebruik momenteel beperkt tot SessionIndexService (zie grep-resultaat in research).
Dependencies geen
U2

Aggregeer worktree-sessies in scanJsonlFiles()

Goal Het project overview toont sessies van zowel de root-cwd als alle worktrees onder dat project.
Files
  • electron/main/services/SessionIndexService.ts — scanJsonlFiles(), getProjectDir()
Approach
  1. Splits getProjectDir(projectPath) in: getEncodedProjectKey(projectPath) (return de string) + getProjectsRoot() (return ~/.claude/projects).
  2. In scanJsonlFiles(): lees readdir(projectsRoot), filter dirs die exact encodedKey matchen of starten met encodedKey + '-worktrees-' of encodedKey + '--worktrees-'.
  3. Parse jsonls uit alle matchende dirs in dezelfde batch-loop. Cache key blijft sessionId (uniek per jsonl-bestand).
  4. Per SessionIndexEntry opslaan welke dir hij vandaan komt (nieuw veld sourceProjectDir?: string — optioneel, alleen voor debug/logging).
  5. Sortering/limit blijft getRecentSessions() (sorts on modified) — geen wijzigingen nodig.
Dependencies U1 (encoding moet kloppen voordat prefix-match werkt)
U3

Strengere worktree-prefix filter (anti-false-positive)

Goal Voorkom dat een ongerelateerd project command-mvp matcht als worktree van command.
Files
  • electron/main/services/SessionIndexService.ts
Approach
  • Match alleen wanneer de suffix start met -worktrees- (Unix) of --worktrees- (Windows met .worktrees) of --claude-worktrees- (Claude Code's eigen worktree-format, zie evidence: C--Users-RemcoVolmer-Code-pascal-ai--claude-worktrees-interesting-knuth-9f27fc).
  • Definieer als constante const WORKTREE_SUFFIX_PATTERNS = ['-worktrees-', '--worktrees-', '--claude-worktrees-'].
Dependencies U2
U4

UI: (optioneel) worktree-badge in sessions-lijst

Goal Gebruiker ziet welke sessies uit een worktree komen, zodat "ik dacht dat dit in main was" niet verwarrend wordt.
Files
  • src/components/ProjectOverview.tsx
  • src/types/index.ts (eventueel worktreeName?: string veld toevoegen)
Approach
  • Toon naast de GitBranch indicator een subtiele label "(worktree)" voor sessies die uit een worktree-dir komen. De gitBranch in de sessie-entry geeft al de branchnaam — dit is verfijning, niet kern.
  • Markeer als defer tenzij U2 evident verwarring oplevert tijdens manual test.
Dependencies U2

5. Test scenarios

U1 — encodeProjectPath (unit, test/sessionIndexService.test.ts)

  • C:\Users\Foo\Code\command → C--Users-Foo-Code-command (baseline, regressie-check)
  • C:\Users\Foo\Code\pascal_ai → C--Users-Foo-Code-pascal-ai (underscore → dash) faalt nu
  • C:\Users\Foo\.claude → C--Users-Foo--claude (dot → dash, dubbele dash blijft) faalt nu
  • C:\Users\Foo\Code\command\.worktrees\fix-x → C--Users-Foo-Code-command--worktrees-fix-x
  • /Users/foo/code/pascal_ai (Unix) → Users-foo-code-pascal-ai (leading dash gestript)
  • Path met spaties: C:\Users\Foo\Code\my project → C--Users-Foo-Code-my-project

U2/U3 — worktree-aggregatie (integration, test/sessionIndexService.test.ts)

  • Mock readdir voor ~/.claude/projects/ die dirs returnt: C--Users-Foo-Code-command, C--Users-Foo-Code-command--worktrees-feat-x, C--Users-Foo-Code-command--worktrees-fix-y, C--Users-Foo-Code-other. Verwacht: bij loadForProject('C:\Users\Foo\Code\command') worden alleen de eerste drie gescand, -other niet.
  • False-positive guard: dir C--Users-Foo-Code-command-mvp mag niet matchen op project command.
  • Lege root-dir maar wel worktree-dirs → sessies uit worktrees verschijnen alsnog (regression check op de huidige early-return bij jsonlFiles.length === 0).
  • Sortering: sessies uit verschillende dirs worden gemengd gesorteerd op modified.

Manual smoke (geen test, voor npm run dev)

  1. Open command project → verwacht meerdere sessies in overview, inclusief van vandaag 1cbfba5f-... en 3c409f99-... Én recente worktree-sessies.
  2. Open pascal_ai project → verwacht 10 sessies, w.o. die van 21 mei.
  3. Klik een worktree-sessie aan → resume werkt zonder errors (sessionId is uniek, terminal start met claude --resume <id>).
  4. Hook-titels uit session-summaries.json verschijnen waar aanwezig (test met sessie 8aad4558-...).

6. Risks & Edge Cases

RiskLikelihoodMitigatie
Claude Code wijzigt encoding in toekomstige versie laag JSDoc bij encodeProjectPath die expliciet verwijst naar empirische bron (lijst ~/.claude/projects/). Tests fixeren de current rule. Versie-noot in SessionIndexService.ts.
Aggregatie scant te veel dirs → trage initial load middel bij power users (100+ projects) readdir in ~/.claude/projects/ is één syscall, dirnamen-filter is O(n) string-match. Batch-parser van 10 jsonls tegelijk blijft de bottleneck en is ongewijzigd.
False-positive prefix-match op vergelijkbare projectnaam middel U3: alleen -worktrees-, --worktrees-, --claude-worktrees- suffix patronen.
Worktree-sessies van inmiddels verwijderde worktrees worden getoond hoog (verwacht) By design — user wil ze juist kunnen resumen. git worktree remove verwijdert geen jsonls. Eventuele cleanup is een aparte feature.
Encoding-fix breekt bestaande caches nihil Cache is in-memory Map per service-instance, geen persistente cache-key. Geen migratie nodig.

7. Verificatie — klaar als

  • npm run test -- test/sessionIndexService.test.ts groen, inclusief nieuwe encoding-tests en aggregatie-tests.
  • Handmatige test op command: minimaal > 2 sessies zichtbaar in overview (er staan tientallen worktree-dirs).
  • Handmatige test op pascal_ai: minimaal 10 sessies zichtbaar.

8. Scope & Defers

In scope

  • Path-encoding match met Claude Code (U1)
  • Worktree-sessie aggregatie in overview (U2-U3)
  • Tests + manual verify

Defer

  • U4 worktree-badge in UI (afhankelijk van manual-test verwarring)
  • Cleanup van weeskinderen-jsonls van verwijderde worktrees
  • "Show sessions from all projects" view
  • Persistente sessie-cache (file-gebaseerd) — nu in-memory en goedkoop genoeg
  • Reverse-encoding helper (van encoded key → cwd) — alleen nodig als we worktree-naam willen tonen, en die kunnen we ook uit gitBranch halen
