Vier ingrepen verankeren kwaliteit in infrastructuur: CI-poort, lint-standaard, gestructureerde logging en één IPC-contract

10 juni 2026 · Origin: docs/brainstorm/2026-06-10-professionalisering-top-5.html (items 1–4; item 5 coverage bewust uitgesteld) · Diepte: Standard · 4 implementation units, 3 PR's

Probleemkader — niets dwingt de bestaande kwaliteit af

De codebase (33k regels, strict TypeScript, 25 testbestanden) heeft géén afdwingmechanisme: er bestaat geen PR-workflow — .github/workflows/build.yml triggert uitsluitend op push naar main en draait alleen een build. Tests, typecheck en lint draaien nergens automatisch. Er is geen ESLint-config (terwijl de eigen workflow-regel die vereist), 120 console-calls verdwijnen in productie in het niets, en het IPC-contract bestaat twee keer (30+ types gedupliceerd in de preload). Dit plan dicht die vier gaten in dependency-volgorde: eerst de standaard (U1), dan de poort die hem afdwingt (U2), daarna logging (U3) en het gedeelde contract (U4) — beide bewaakt door de poort uit U2.

Technische kernbeslissingen

BeslissingRationale
ESLint 9 flat config met typescript-eslint recommended — niet type-checked als startpuntType-checked linting op 33k regels geeft honderden meldingen en blokkeert de uitrol. Recommended + no-explicit-any (error) + react-hooks + consistent-type-imports borgt de huidige kwaliteit direct; aanscherpen naar type-checked is een latere, losse stap.
Nieuwe, aparte ci.yml op pull_request — build.yml blijft ongemoeidbuild.yml doet release-artefacten op push naar main; dat is een ander doel dan een merge-poort. Drie parallelle jobs (lint, typecheck, test) geven losse, snelle PR-checks. Minimal impact: bestaand workflow-gedrag verandert niet.
Typecheck als los script tsc --noEmitNu zit tsc alleen in het build-script gekoppeld aan vite + electron-builder; als losse check faalt hij apart, sneller en zonder artefact-bijwerkingen.
electron-log (v5) voor main-process logging; Sentry uitgesteldConform brainstorm: pragmatisch starten met lokale logs die een gebruiker kan opsturen. electron-log is drop-in, schrijft naar userData met rotatie, en vereist geen account/DSN/privacy-afweging. Sentry blijft een losse vervolgbeslissing.
Gedeelde types in shared/, geïmporteerd met import typeDe preload-comment ("cannot import from src/") geldt voor runtime-imports; type-only imports worden door de bundler volledig geërased, dus procesisolatie en de cjs-preload-bundle blijven intact. consistent-type-imports (U1) dwingt af dat het type-only blíjft.
Unit-tests in CI op windows-latest + ubuntu-latest; e2e niet in deze faseWindows is het primaire platform; ubuntu is de snelle vangrail. build.yml bewijst dat node-pty op beide runners compileert. De Playwright-smoketest vereist een display en is flaky-gevoelig — uitgesteld naar item #5 (coverage-fase).

Technisch ontwerp — volgorde en afhankelijkheden

PR-indeling:

  • PR 1 — U1 + U2 samen (conform brainstorm): de lint-job heeft de config nodig; samen voorkomt dit een tussenstand waarin CI een niet-bestaand script aanroept.
  • PR 2 — U3: logging, los te reviewen (raakt 17 service-bestanden mechanisch).
  • PR 3 — U4: gedeeld contract, los te reviewen (verplaatst types, geen gedragswijziging).

U3 en U4 kunnen parallel; ze raken verschillende bestanden (services vs. types/preload).

Implementation units

U1.Lint-standaard: ESLint flat config + Prettier + losse quality-scripts

Codifieert de impliciete standaard zodat U2 hem kan afdwingen. Advances R2.

Bestanden

eslint.config.js (nieuw).prettierrc (nieuw).prettierignore (nieuw)package.json (scripts + devDeps)

Aanpak

  • DevDeps: eslint, typescript-eslint, eslint-plugin-react-hooks, prettier, eslint-config-prettier.
  • Flat config over src/, electron/ en test/; ignores voor dist*/, release/, docs/, node_modules/.
  • Regels: typescript-eslint recommended; no-explicit-any op error met één gedocumenteerde disable-regel op de bestaande Milkdown-any in src/components/Editor/MarkdownEditor.tsx; react-hooks recommended; consistent-type-imports (nodig voor U4); eslint-config-prettier als laatste om conflicten uit te zetten.
  • Scripts: lint, lint:fix, format, typecheck (= tsc --noEmit).
  • Eenmalige pass: lint:fix + format over de hele codebase; resterende meldingen handmatig oplossen of per regel beargumenteerd op warn — de PR merget pas als lint exit 0 geeft.

Testscenario's

  • Test expectation: none — pure tooling-config. Verificatie is gedragsmatig: (a) npm run lint exit 0 op de hele codebase; (b) een bewust toegevoegde const x: any in een scratch-bestand geeft een error; (c) npm run typecheck exit 0; (d) de bestaande testsuite draait ongewijzigd groen (autofix mag geen gedrag raken).

U2.CI-kwaliteitspoort: PR-workflow met drie parallelle blocking checks

Maakt falende tests, types of lint zichtbaar en blokkerend vóór merge. Advances R1. Depends: U1.

Bestanden

.github/workflows/ci.yml (nieuw)

Aanpak

  • Trigger: pull_request naar main + push naar main (de poort vangt ook directe pushes).
  • Drie parallelle jobs: lint (ubuntu, npm run lint), typecheck (ubuntu, npm run typecheck), test (matrix: windows-latest + ubuntu-latest, npm test — pretest draait de vite test-build automatisch mee).
  • npm ci + Node 20 + npm-cache via actions/setup-node; concurrency-group per branch zodat verouderde runs gecanceld worden.
  • build.yml blijft ongewijzigd (release-artefacten op push main).
  • Operationele stap (handmatig, eenmalig): branch protection op main aanzetten met de drie checks als required — kan niet vanuit de repo zelf.

Testscenario's

  • Test expectation: none — workflow-config. Verificatie op de PR zelf: (a) de drie checks verschijnen los in de PR-status en zijn groen; (b) een bewust gebroken test in een fixup-commit maakt de test-job rood (daarna revert); (c) test-job slaagt op beide OS'en incl. node-pty-compilatie; (d) na merge: branch protection geactiveerd en zichtbaar op main.

U3.Gestructureerde logging: electron-log vervangt 120 console-calls, stille fouten worden zichtbaar

Maakt productieproblemen diagnosticeerbaar via een logbestand dat een gebruiker kan opsturen. Advances R3. Depends: U2 (CI bewaakt de refactor).

Bestanden

electron/main/services/Logger.ts (nieuw, dunne wrapper)electron/main/index.tselectron/main/services/*.ts (17 bestanden, mechanisch)electron/main/services/ClaudeHookWatcher.tselectron/main/services/CommandServer.tstest/logger.test.ts (nieuw)

Aanpak

  • electron-log v5 als devDep→dep; init in main: file-transport naar userData met rotatie (max ~5MB), console-transport alleen in dev; niveau via env overrulebaar.
  • Dunne Logger.ts-wrapper met scoped loggers (log.scope('TerminalManager')) — services importeren de wrapper, niet electron-log direct, zodat de transport-keuze één plek heeft en tests kunnen mocken.
  • Mechanische vervanging van console.* in electron/main/ met behoud van de bestaande [ServiceName]-prefixes als scope. Renderer blijft buiten scope.
  • De 5 stille catches: WorktreeService.ts:212 (git pull) en AutomationRunner.ts:425 (kill) krijgen een debug/warn-log; de 3 listener-callback-catches (ClaudeHookWatcher, FileWatcherService, GitHubService) loggen op warn met throttling-overweging (callbacks vuren frequent).
  • ClaudeHookWatcher: state-write via temp+rename, identiek aan het bestaande patroon in ProjectPersistence.ts.
  • CommandServer: één request-log-regel (methode, pad, status, duur) op debug-niveau.
  • Bestaande IPC app:open-crash-log uitbreiden of spiegelen zodat ook het reguliere logbestand te openen is vanuit de UI (Settings).
  • Execution note: vervang console-calls per service in kleine commits; de bestaande servicetests draaien na elke stap.

Testscenario's

  • Logger-wrapper: scoped logger schrijft regel met scope-prefix en niveau naar de file-transport (mock transport, assert formaat).
  • Logniveau-filtering: debug-call verschijnt niet wanneer niveau op info staat.
  • ClaudeHookWatcher atomic write: na een gesimuleerde crash tussen temp-write en rename is het oorspronkelijke statebestand intact (bestaande test in claudeHookWatcher.test.ts uitbreiden).
  • Stille-catch-fix: een listener-callback die throwt produceert een warn-log en stopt de overige listeners niet (uitbreiding bestaande watcher-tests).
  • CommandServer: een afgehandelde request produceert exact één debug-regel met status en duur (uitbreiding commandServer.test.ts).
  • Regressie: volledige bestaande suite groen — de vervanging is gedragsneutraal.

U4.Eén IPC-contract: gedeelde types in shared/, preload-duplicatie verwijderd

Type-drift tussen preload en renderer wordt een compile-fout in plaats van een runtime-bug. Advances R4. Depends: U2 (typecheck-poort vangt de refactor).

Bestanden

shared/ipc-types.ts (nieuw)src/types/index.tselectron/preload/index.tselectron/main/index.ts + services (imports)tsconfig.json (include + path-alias)eslint.config.js (shared/ in scope)

Aanpak

  • Verplaats de contract-types die de procesgrens kruisen (Project, ProjectSettings, Worktree, GitStatus, AccountProfile, TerminalState, SpawnFailedPayload, etc. — de ~30 nu gedupliceerde definities) naar shared/ipc-types.ts. Renderer-only types blijven in src/types/index.ts, dat re-exporteert vanuit shared zodat bestaande imports blijven werken.
  • Preload vervangt alle lokale definities door import type uit shared/; de bestaande sync-NOTE-comments (o.a. regel 8–9 en 52) vervallen.
  • Type-only is de harde randvoorwaarde: geen runtime-waarden in shared/ipc-types.ts (geen consts, geen functies), afgedwongen door consistent-type-imports uit U1.
  • tsconfig.json: "shared" toevoegen aan include.
  • Patroon volgen: de bestaande @/*-alias in tsconfig + vite.config; voeg analoog een @shared/*-alias toe in beide.

Testscenario's

  • Drift-bewijs (kern van de unit): een veld toevoegen aan een shared type en alleen renderer-side gebruiken compileert; een veld hernoemen breekt zowel preload- als renderer-compile — handmatig geverifieerd vóór merge, daarna bewaakt door de typecheck-job.
  • Bundle-verificatie: dist-electron/preload/index.cjs bevat na build geen code uit shared/ of src/ (types volledig geërased) en de app start met werkende IPC.
  • Regressie: volledige bestaande suite groen; npm run build levert een startbare app (handmatige smoke: project openen, chat starten, git-panel).

Risico's en mitigatie

RisicoMitigatie
ESLint-uitrol produceert een grote, lastig reviewbare diff (autofix over 133 bestanden).Autofix/format in één losse commit zonder handmatige wijzigingen; inhoudelijke fixes in aparte commits. Reviewer beoordeelt de config, niet de mechanische diff.
node-pty-compilatie faalt in de CI test-job.Laag: build.yml bewijst al maanden dat npm install + node-pty werkt op windows/ubuntu runners. Bij falen: npm ci-cache check, zelfde Node 20.
Console-vervanging (U3) wijzigt onbedoeld gedrag in een service.Per-service commits; vervanging is 1-op-1 (zelfde call sites, zelfde argumenten); volledige suite na elke service.
Preload-bundle trekt per ongeluk runtime-code uit shared/ binnen (U4).consistent-type-imports + afspraak "geen waarden in shared/" + bundle-inspectie als expliciet testscenario.
Branch protection vergeten → poort bestaat maar blokkeert niets.Expliciete operationele stap in U2-verificatie; pas afvinken als een test-PR de blokkade aantoont.
